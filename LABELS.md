# GitHub Labels Strategy

To manage SerapeumAI professionally, use these standard labels to categorize issues and pull requests. You can create these manually in the **Settings > Labels** tab of your GitHub repository.

## 🎨 Categorization
- **area:ui** (Color: `#fbca04`) - Issues related to the UI/UX.
- **area:engine** (Color: `#1d76db`) - Issues related to the AI extraction or logic.
- **area:infra** (Color: `#6e4cc2`) - Issues related to databases, config, or install scripts.

## 🐞 Types
- **bug** (Color: `#d73a4a`) - Something isn't working.
- **enhancement** (Color: `#a2eeef`) - New feature or request.
- **documentation** (Color: `#0075ca`) - Improvements to MD files.
- **wontfix** (Color: `#ffffff`) - This will not be worked on.

## 🤝 Community & Help
- **good first issue** (Color: `#7057ff`) - Simple issues for new contributors.
- **help wanted** (Color: `#008672`) - Extra attention is needed.
- **question** (Color: `#d876e3`) - Further information is requested.

## 🚦 Workflow
- **priority:high** (Color: `#b60205`) - Needs immediate attention.
- **in-progress** (Color: `#cccccc`) - Currently being worked on.
- **on-hold** (Color: `#e99695`) - Waiting for more info or blocked.
