# 02 — Scope and Non-Scope

## Purpose

This document protects the Total Quality Upgrade from becoming an uncontrolled redesign.

## In scope

### Documentation governance

- GitHub truth reset.
- README and roadmap truth alignment.
- Public/internal documentation separation.
- Changelog and limitation discipline.
- Documentation impact gate for every packet.

### Quality foundation

- Quality status vocabulary.
- Source-lane taxonomy.
- Extension support matrix.
- User-visible support and limitation labels.
- Quality report contract.

### Extraction and evidence quality

- PDF pre-inspection and page classification.
- OCR-needed detection and OCR quality reporting.
- Drawing evidence quality improvements.
- IFC/BIM hierarchy/property evidence.
- P6/XER schedule graph quality.
- Excel/register extraction quality.
- DOCX/PPTX structure preservation.
- Metadata extraction quality.

### Document Center quality

- Four-tab review model.
- Deterministic extraction separated from AI/VLM output.
- Metadata visibility.
- Honest empty states and source-lane labels.

### Chat quality

- Active-project binding.
- Snapshot/truth-scope awareness where applicable.
- Certified-facts-first answer path.
- Honest refusal.
- Evidence availability.
- No candidate facts as governing truth.
- No LLM calculations.

### Agentic workflow spine

- Tool registry.
- Tool policy gate.
- Skill registry.
- Agent run state.
- Safe trace.
- Memory separation.
- ToolBench / AgentBench.

### Gold fixtures and proof

- Non-confidential, redistributable, small gold fixtures.
- Expected outputs.
- Regression runner.
- Quality-dimension failure reporting.

### Controlled labs

- Bounded review swarm lab.
- MCP / external connector lab.
- QuantBench and model fit matrix.
- Local 3D review concepts lab.

## Out of scope

- Packaging rewrite.
- Dependency upgrades without explicit approval.
- New installer/distribution model.
- Cloud-required workflows.
- Revit replacement or CAD/BIM authoring.
- Guaranteed compliance claims.
- Autonomous submissions, approvals, or file edits.
- Uncontrolled agent swarms.
- Arbitrary MCP execution in the baseline.
- Web access by default.
- Memory as certified truth.
- AI/VLM output as deterministic evidence.
- Public hosted 3D tools with project files.
- Spatial scene edits as certified BIM truth.

## Scope control rule

Every packet must define allowed scope, forbidden scope, likely touched files, documentation impact, risk frame, and exit condition before implementation.
