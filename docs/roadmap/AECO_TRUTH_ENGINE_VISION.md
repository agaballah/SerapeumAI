["ROADMAP_OVERVIEW.md`](ROADMAP_OVERVIEW.md)\n\n---\n\n## TL;DR FOR THE TEAM\n\nWe are NOT building \"chat with files", ".", "We are building an OFFLINE ENGINEERING TRUTH SYSTEM:\n\n- Ingest engineering artifacts (IFC/BIM, P6, PDFs, Office, emails, logs).\n- Run deterministic extractors \u2192 structured staging tables with provenance.\n- Run deterministic fact builders \u2192 explicit computed facts with lineage.\n- Validate/certify facts via rules + human queue.\n- LLM is a query planner + narrator. It can answer ONLY from CERTIFIED facts.\n- If facts are missing, LLM must refuse and output a coverage gap + job plan.\n\n**Non-negotiable:** If it's not in certified facts with citations, we do not answer.\n\n---\n\n## 1) DEFINITIONS (STRICT SEMANTICS)\n\n### 1. Evidence\nAtomic extracted item tied to a specific file version and location.\nExamples: OCR span with bbox, PDF text block, Excel cell, P6 activity record,\n          IFC property, email header field.\n\n### 2. Extraction\nStructured output of an extractor: a set of evidence records + metadata + diagnostics.\nAlways reproducible from the file version + extractor version.\n\n### 3. Canonical Entity\nNormalized \"real project object", "across sources.\nExamples: Drawing (by drawing number), Document revision, Activity, IFC element GUID,\n          Equipment tag, RFI, Submittal, Permit.\n\n### 4. Crosswalk Link\nExplicit link between canonical entities.\nExamples: Activity \u2192 Zone, Activity \u2192 Elements, RFI \u2192 Elements, Submittal \u2192 Equipment.\nLinks can be VALIDATED or CANDIDATE; assistant can only rely on VALIDATED links.\n\n### 5. Fact (Ground Truth Unit)\nA computed/validated statement allowed to be used in an answer.\nMust include: fact_type, subject_id, scope, as_of snapshot, typed value, method version,\n             status, and lineage to evidence + links.\n\n### 6. Snapshot (As-Of State)\nA coherent \"project state", "that answers refer to:\n- schedule snapshot (data date, baseline)\n- design snapshot (model revision/issue package)\n- docs snapshot (latest revisions set)\n- registers snapshot (RFI/submittal/procurement/field logs cut-off)\n\n### 7. Certified Facts\nFacts with status=VALIDATED (automated rule pass) and if required, HUMAN_CERTIFIED.\n\n---\n\n## 2) PRODUCT GUARANTEES (ACCEPTANCE CRITERIA)\n\n### G1. Strictness\nThe assistant must refuse any claim that cannot be fully supported by certified facts.\n\n### G2. Reproducibility\nGiven same file versions + extractor/builder versions, the DB can be rebuilt and match.\n\n### G3. Provenance\nEvery answer must cite facts, and every fact must cite evidence locations.\n\n### G4. Version correctness\nFacts are computed \"as-of", "an explicit snapshot; superseded facts remain in history.\n\n### G5. Conflict safety\nIf conflicting certified facts exist, assistant must disclose conflict; no silent choice.\n\n---\n\n## 3) ARCHITECTURE (OFFLINE)\n\n### 3.1 Components\n\n#### A) Windows Desktop UI\n-
